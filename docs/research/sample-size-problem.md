# サンプルサイズ問題

個人の経験ログから「この仮説は正しい」と言えるためには、どれだけのデータが必要か。

---

## 問題の本質

「何回経験すれば確かめられるか」に対する答えは一意には出ない。必要なサンプルサイズは以下によって変わる：

- **求める信頼度**：95%確信したいのか、70%でいいのか
- **ばらつきの大きさ**：結果が毎回安定しているか、バラバラか
- **効果の大きさ**：仮説の影響が大きければ少ない回数でも検出できる

## 個人経験の追加的な困難

統計的なサンプルサイズ計算が成立しても、個人の経験ログにはさらなる問題がある：

| 問題 | 内容 |
|---|---|
| 測定のブレ | 体調・環境・測定方法が毎回異なる |
| 交絡変数 | 他の要因が結果に影響している可能性 |
| 確証バイアス | 信じたい結果を記録しやすい |

## このアプリでの結論

**「保証」はできない。**

どれだけ回数を積み重ねても、個人の経験ログから仮説の正しさを統計的に保証する閾値は定まらない。

→ ポパーの反証主義と同じ立場をとる：知識は「証明されたもの」ではなく「**現時点で反証されていないもの**」として扱う。（[knowledge-verification-theory.md](knowledge-verification-theory.md) 参照）

## 長期ログ蓄積の可能性と限界

経験ログが蓄積されれば、Claudeの定性判断に加えて頻度ベースの補正も可能になる。

ただし：
- あくまで「自分の中での傾向」であり、一般化はできない
- 閾値（何回で十分か）は設計上の判断として決める必要がある

**→ 未解決の設計判断：`verified` への閾値をどう決めるか**（[knowledge-verification-theory.md § 5](knowledge-verification-theory.md) 参照）
