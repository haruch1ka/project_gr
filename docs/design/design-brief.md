# Design Brief — project_gr

## アプリ概要

「あらゆる物事の上達を、Web知識と実経験の統合によって加速するツール」。

ユーザーが釣り・筋トレなどの分野を自由に定義し、Web収集した情報（仮説）と自分の経験ログをAIが統合・体系化する。知識は確信度スコアで管理され、経験の積み重ねにより `hypothesis → verified / disproved` へと遷移する。

**ターゲット：** 特定分野を本気で上達させたい個人（20〜40代）。記録を続けることが目的ではなく、上達を実感できることが目的。

---

## 主要フロー

```
① Web情報収集（手動トリガー）
   断片情報を取得 → 「仮説」として知識プールに投入

② 経験ログ記録（軽量・摩擦ゼロ）
   仮説を意識しながら実践 → 一言で記録

③ AI対話による統合（重量・深い問い）
   経験 × Web情報 × 知識プールを文脈に
   対話後にアクション選択肢を提示
     [ プラン提案 / グラフ表示 / 知識保存 ]

④ 知識が育つ
   hypothesis → verified / disproved
   信頼度スコアが経験で上下

⑤ 体系化された知識 → 次の行動プランへ → ①に戻る
```

---

## UX 原則

| モード | 場面 | 目標 |
|--------|------|------|
| 軽量モード | 経験ログ・アンケート | 摩擦ゼロ・5秒以内で完結 |
| 重量モード | AI対話 | コストをかけて深く考える |

- 入力は「意思・感情・気づき」のみ。構造化はAIが担う
- 「一言入力 → AIが構造化 → ユーザーが確認タップ」が基本パターン

---

## デザイントーン

- **ミニマル** — 余計なものを置かない。コンテンツに集中させる
- **知識が育つ感覚** — confidenceScore の上昇・status の変化を可視化して達成感を与える
- **落ち着いた暗め** or **クリーンな白** — 長時間見ても疲れない配色

## 画面一覧

### 1. 経験ログ（log）— 最優先

**目的：** 経験をゼロ摩擦で記録する

**フロー：**
1. 分野タップ（既存分野の一覧 or ＋で新規）
2. 日付（デフォルト今日）
3. 一言メモ入力（テキストエリア、大きめ）
4. 完了ボタン → 「AIが構造化します」トースト → 確認タップ

**重要なUI要素：**
- 分野選択は大きなチップ/カード形式（タップしやすい）
- メモ入力欄が画面の主役（余計な項目を置かない）
- 所要時間：5秒以内を目指す

**レイアウトヒント：**
```
┌─────────────────────────┐
│  経験を記録              │  ← ヘッダー（小さく）
├─────────────────────────┤
│  分野を選ぶ              │
│  ┌──────┐ ┌──────┐ ┌──┐│
│  │ 釣り  │ │筋トレ│ │＋││  ← 大きなチップ
│  └──────┘ └──────┘ └──┘│
├─────────────────────────┤
│  2026-06-04  ▼          │  ← 日付（デフォルト今日）
├─────────────────────────┤
│                          │
│  今日どうだった？         │  ← メモ入力（画面の主役）
│  （大きなテキストエリア）  │
│                          │
│                          │
├─────────────────────────┤
│      ［ 記録する ］       │  ← 主要CTA
└─────────────────────────┘
```

### 3. 知識一覧・詳細（knowledge）

**目的：** 蓄積された知識を確認・信頼度を把握する

**一覧：**
- カード形式（タイトル・status バッジ・confidenceScore バー）
- フィルタ：status / category / field

**詳細：**
- content（知識の内容）
- confidenceScore バー（視覚的に）
- status バッジ（hypothesis=グレー / verified=グリーン / disproved=レッド）
- 裏付けた経験・反例の経験（折りたたみ）
- 元になったWeb情報（折りたたみ）

**レイアウトヒント（一覧）：**
```
┌─────────────────────────┐
│  知識  [全て▼] [釣り▼]  │  ← フィルタ
├─────────────────────────┤
│  ┌───────────────────┐  │
│  │[verified] ルアーの色 │  │
│  │▓▓▓▓▓▓▓░░░  72%    │  │  ← confidenceScore バー
│  └───────────────────┘  │
│  ┌───────────────────┐  │
│  │[hypothesis] 朝マヅメ│  │
│  │▓▓░░░░░░░░  23%    │  │
│  └───────────────────┘  │
└─────────────────────────┘
```

**レイアウトヒント（詳細）：**
```
┌─────────────────────────┐

### 4. AI 対話（dialogue）

**目的：** 経験と知識をAIと深く対話して統合する

**UI：**
- チャット形式（吹き出し）
- 対話完了後にアクション選択肢を表示：
  - [ プランを提案してもらう ]
  - [ 知識として保存する ]
  - [ グラフで可視化する ]

**レイアウトヒント：**
```

│  AI対話  [釣り]          │
├─────────────────────────┤
│        ┌──────────────┐ │
│        │ 最近どんな経験 │ │  ← AI（右寄せ吹き出し）

│  ┌──────────────┐       │

## データモデル

```typescript
// 経験ログ
Experience {
  field: string          // ユーザーが自由定義（例: "釣り"）
  date: Date
  memo: string           // 自由記述
  createdAt: Date
}

// 統合知識
Knowledge {
  field: string
  category: string       // ユーザー定義のカテゴリ
  content: string
  webSources: ResearchResult[]
  supportingExperiences: Experience[]    // 裏付けた経験
  contradictingExperiences: Experience[] // 反例になった経験
  confidenceScore: number  // 0.0〜1.0、ベイズ的に更新
  status: 'hypothesis' | 'verified' | 'disproved'
  tags: string[]
}

// 行動プラン
Plan {
  field: string
  proposal: string       // AIの提案テキスト
  dialogHistory: ChatMessage[]
  reviewedAt: Date | null
  reviewNote: string | null
  createdAt: Date
}

// Web収集生データ
ResearchResult {
  field: string
  query: string
  results: { title: string; url: string; snippet: string }[]
  collectedAt: Date
  usedInKnowledgeIds: string[]
}
```

**confidenceScore の意味：**
- 0.0〜0.3：hypothesis（仮説）— グレー表示
- 0.3〜0.7：途中（まだ仮説）
- 0.7〜1.0：verified（現時点で反証されていない）— グリーン表示
- disproved：反例が蓄積されたもの — レッド表示
